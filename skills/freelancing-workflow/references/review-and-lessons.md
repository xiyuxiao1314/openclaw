# Review and Lessons

## Review Goal

Capture what happened on a specific project.

Questions:
- Why was the task accepted or rejected?
- Did scope drift occur?
- Did the client show warning signs?
- Was the original estimate fair?
- What judgment was correct?
- What judgment was wrong?

## Review Record Template

- Project title:
- Type:
- Accepted / rejected:
- Why:
- Scope drift: yes / no
- Client warning signs:
- Outcome:
- What worked:
- What failed:
- What should change next time:

## Lesson Goal

Convert one project’s experience into reusable system improvements.

## Lesson Template

- Project type:
- Pattern observed:
- Future handling rule:
- Where the workflow should change:
- Should this be added to the skill: yes / no

## Rule

A project is not fully finished until at least one reusable lesson is considered, even if the answer is “no change needed.”
