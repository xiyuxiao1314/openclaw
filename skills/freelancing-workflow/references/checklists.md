# Checklists

## Execution Pre-Check

- [ ] task is small
- [ ] goal is clear
- [ ] deliverable is clear
- [ ] documents were read
- [ ] missing details were clarified
- [ ] no difficult bug is hiding underneath
- [ ] no high-risk permission or production action is required
- [ ] a clear path exists to complete the work
- [ ] the user approved proceeding
- [ ] scope and timing were not overpromised

## Delivery Pre-Check

- [ ] requested work is complete
- [ ] no core item was missed
- [ ] result matches the agreed scope
- [ ] no unverified guess is presented as fact
- [ ] basic self-check was done
- [ ] no sensitive information is exposed
- [ ] no policy or safety boundary was crossed
- [ ] the user approved final delivery

## Aftercare Check

- [ ] issue belongs to original scope
- [ ] this is a fix, not a new feature
- [ ] the task nature has not changed
- [ ] support remains bounded and reasonable

## Review Questions

- Why was this task accepted or rejected?
- Did scope drift happen?
- Did the client show warning signs?
- Was the original estimate fair?
- What should be tightened next time?
