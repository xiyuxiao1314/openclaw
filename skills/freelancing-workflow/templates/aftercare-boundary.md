# Aftercare Boundary Template

## Purpose

Use this when a client comes back after delivery and you need to separate a reasonable fix from a new request.

## Message Template

Hello, I reviewed your follow-up.

I can help with issues that fall within the original agreed scope. Based on what you described, this appears to be:

- within the original scope
or
- outside the original scope and closer to a new request

If it is within the original scope, I can address it as a bounded follow-up.
If it goes beyond the original scope, we should define it separately before proceeding.

## Rule

Do not treat new features, rewritten deliverables, or changed task goals as aftercare.
